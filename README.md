# 📡 Reactive Streaming

A reactive Spring Boot application for real-time telemetry data streaming, built with **Project Reactor**, **Spring WebFlux** and **Reactive MongoDB**.

---

|---|---|
| Java | 25 |
| Spring Boot | 3.5.11 |
| Spring WebFlux | bundled with Spring Boot |
| Spring Data MongoDB Reactive | bundled with Spring Boot |
| Project Reactor | bundled with Spring Boot |

## 🔌 API Endpoints

### POST `/api/telemetry/ingest`

Saves a telemetry event to the database.

- **Content-Type:** `application/json`
- **Body:**

```json
{
  "deviceId": "device-001",
  "temperature": 23.5,
  "timestamp": "2024-01-15T10:30:00Z"
}
```

> The `timestamp` field is optional: if omitted, it is automatically set to the current time.

- **Response:** `200 OK` with the saved event (including the `id` generated by MongoDB)

---

### GET `/api/telemetry/stream/{deviceId}`

Opens a **Server-Sent Events (SSE)** stream for a specific device.

- **Accept:** `text/event-stream`
- **Path variable:** `deviceId` — device identifier

- **Response:** continuous stream of events in SSE format

## 🚀 Running the Application

### Prerequisites

- JDK 25+
- Maven 3.9+ or use the included wrapper `mvnw`
- MongoDB running (default: `localhost:27017`)

### Run

```bash
./mvnw spring-boot:run
```

On Windows:

```powershell
.\mvnw.cmd spring-boot:run
```

### Build

```bash
./mvnw clean package
java -jar target/reactive-streaming-0.0.1-SNAPSHOT.jar
```

## 🧪 Manual Testing with curl

### 1. Ingest an event (with timestamp)

```bash
curl -X POST http://localhost:8080/api/telemetry/ingest \
  -H "Content-Type: application/json" \
  -d '{"deviceId":"device-001","temperature":23.5,"timestamp":"2024-01-15T10:30:00Z"}'
```

### 2. Ingest an event (without timestamp — auto-generated)

```bash
curl -X POST http://localhost:8080/api/telemetry/ingest \
  -H "Content-Type: application/json" \
  -d '{"deviceId":"device-001","temperature":25.0}'
```

### 3. Open the SSE stream

```bash
curl -N -H "Accept: text/event-stream" \
  http://localhost:8080/api/telemetry/stream/device-001
```

## 🧪 Manual Testing with PowerShell

### Ingest

```powershell
Invoke-RestMethod -Method POST `
  -Uri "http://localhost:8080/api/telemetry/ingest" `
  -ContentType "application/json" `
  -Body '{"deviceId":"device-001","temperature":23.5,"timestamp":"2024-01-15T10:30:00Z"}'
```

### SSE Stream

```powershell
Invoke-WebRequest -Uri "http://localhost:8080/api/telemetry/stream/device-001" `
  -Headers @{"Accept"="text/event-stream"} `
  -Method GET
```

## 🔁 End-to-End Test Flow

1. Start MongoDB locally (or via Docker: `docker run -d -p 27017:27017 mongo`)
2. Start the application with `mvnw spring-boot:run`
3. In one terminal, open the **stream** for `device-001`
4. In a second terminal, send multiple **ingest** requests for the same `deviceId`
5. Verify that events appear in real time in the stream terminal
